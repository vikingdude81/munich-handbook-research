## 2. The Magic Circle — `src/lgi_manifold.py`

**The math does not do what the docstring says, and at current data scale the
"circle" is vacuous.** Four distinct problems:

1. **Docstring/implementation mismatch.** The docstring defines the circle as an
   integer lattice L = {Σ aᵢvᵢ | aᵢ ∈ ℤ} and membership as "expressible as an
   integer linear combination." The code computes a QR orthogonal basis and
   projects onto the **real linear span** — the integer constraint is never
   enforced. A span is not a lattice.
2. **Vacuous membership test.** With 1,111+ attributed spirit vectors in 28
   dimensions, the span is all of ℝ²⁸. Every vector projects onto itself,
   `residual()` ≈ 0 for anything, and `is_within_circle()` is **always True**.
   The Magic Circle admits everything — which is thematically ironic and
   mathematically useless. (It was written for a ~4-spirit seed set, where the
   span was a genuine 4-D subspace of ℝ²⁸.)
3. **`radius=7.0` is arbitrary** — residuals live in [0, ~1] after feature
   normalization, so even a genuine subspace test would never exceed the radius.
4. **`shortest_vector_approx` is 5ⁿ brute force** over n = number of spirits —
   astronomically infeasible beyond ~10 spirits. `lattice_volume()` takes the
   Gram determinant of a rank-deficient 1111×1111 matrix → 0/numerical noise.

**Recommendation:** either (a) re-scope it explicitly to a small curated seed set
(≤10 archetypal spirits) where the subspace/lattice math is meaningful, or
(b) replace the membership test with something that scales: Mahalanobis distance
to the spirit distribution, distance to the top-k PCA subspace, or convex-hull /
one-class-SVM membership. Option (a) is truest to the "circle inscribed with few
chosen names" metaphor.

## 3. The HTML apps (live-tested via `python -m http.server`)

### 3a. `munich_handbook_app.html` — ✅ WORKS
- All 5 tabs (Circle / Spirits / Courts / Resonances / Hours) function; zero
  console or runtime errors; planetary-hours tab renders time-dependent content.
- Data is a **hardcoded, curated 12-spirit subset** (Astaroth, Belial, Paymon,
  Berith, …) — deliberately "verified provenance only," which is honest. It is
  NOT wired to `unified_entities.json`.
- Stale claim: footer says "126 chunks" — that was the original 3-source ingest
  (39+39+48); the corpus is now 160 chunks across 5 ingests.

### 3b. `draper_sphere_v2.html` — ❌ was COMPLETELY BROKEN → ✅ FIXED
- The page rendered **blank**: two SVG `<filter>` elements (ids `gl`, `glB`)
  were never closed, so the in-browser Babel JSX compile failed and React never
  mounted. No visible error — just an empty page.
- **Fixed** by closing both tags. Verified: full render (zodiac wheel, planets,
  Kircher organ correspondences) and all 9 controls (Draper 1561 / Today /
  Custom / Sphere / Reading / Bruno / Angels / Hours / Cipher) work with zero
  errors.
- Note: depends on **unpkg CDN** for React + Babel standalone → needs internet,
  and in-browser Babel compiles ~48K of JSX on every load. Fine for research;
  precompile if it ever ships.

### 3c. `porta_cipher_wheel.html` — ✅ WORKS, cipher verified correct
- All tabs work; zero errors.
- **Cipher correctness verified programmatically**: Moon key (shift 13)
  encrypts "CONJURO TE ASTAROTH" → "PBAWHEB GR NFGNEBGU" = exact ROT13 ✓;
  Saturn key displays the Atbash mirror alphabet (A↔Z, B↔Y…) matching its
  "retrograde principle" description ✓. Agrippa correspondences (Cassiel/Zazel,
  lead, etc.) are standard.
- **Historical nitpick**: the app titles the source "De Occultis Literarum
  Notis (Naples, 1592)". Della Porta's cryptography treatise was published as
  *De Furtivis Literarum Notis* (Naples, **1563**); *De Occultis Literarum
  Notis* is the title of the later reissue (c. **1593**, unauthorized edition).
  "1592, Naples" matches neither edition cleanly — check the scanned source
  edition and align the date/title.

### Shared app caveats
- All three embed their data as constants — they don't read the pipeline's DB,
  so DB corrections don't propagate. Acceptable by design (self-contained,
  curated), but the curation step should be documented.
- All three load fonts/libs from CDNs — offline use shows unstyled/blank pages.
